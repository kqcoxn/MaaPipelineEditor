## 简介
本文件面向 MaaPipelineEditor 的"布局算法"能力，系统化梳理自动布局、对齐吸附、智能间距调整、手动与自动布局切换、布局保存与恢复、以及配置项与性能优化策略。文档以代码为依据，结合可视化图示，帮助开发者与使用者理解并高效使用布局功能。特别关注新增的局部自动布局功能，该功能允许用户对选定节点进行智能定位，而非整个流程图。

## 核心组件
- 自动布局（ELK）
  - 使用 ELKJS 的分层布局算法，支持全局自动布局和局部自动布局两种模式
  - 全局模式：对整个流程图进行智能定位
  - 局部模式：仅对选定节点应用智能定位，同时保留其他节点位置不变
  - 关键点：节点需具备测量尺寸；空图直接返回；异常捕获；异步执行。
- 对齐吸附
  - 提供网格/边缘/中心对齐的六种模式：Left、Right、Top、Bottom、Center、Middle；拖拽过程实时计算吸附偏移与参考线；支持仅在可视范围内吸附。
- 智能间距调整
  - 基于选中节点集合，以基准点为参照，按距离比例分配位移，实现"整体均匀扩展/压缩"的视觉效果。
- 手动与自动布局切换机制
  - 工具栏按钮根据选中节点数量智能切换：2个及以上节点时显示"局部自动布局"，否则显示"自动布局"
  - 选择上下文菜单提供完整的布局选项，包括局部自动布局和全局自动布局
- 布局保存与恢复
  - 通过状态替换接口批量写入节点位置；历史记录保存由状态切片统一管理；粘贴时自动适配坐标与分组关系。
- 配置项
  - 磁吸开关与仅可视范围吸附；吸附阈值与参考线样式；自动布局方向、间距等参数在 ELK 配置中集中定义。

## 详细组件分析

### 局部自动布局（autoPartial 方法）
- 功能概述
  - 新增的局部自动布局功能，允许用户对选定节点应用智能定位算法
  - 仅对选中节点之间的边进行布局计算，忽略外部连接
  - 保持其他节点位置不变，仅调整选中节点的相对位置
- 输入输出
  - 输入：选中的节点数组（至少2个节点）
  - 输出：更新后的节点位置，通过状态替换应用到画布
- 核心特性
  - 边缘过滤：仅保留选中节点之间的边，过滤掉外部连接
  - 包围盒计算：记录选中节点的原始包围盒，确保布局后的位置关系保持一致
  - 坐标空间映射：将布局结果偏移到原始包围盒位置，保持整体布局的稳定性

```mermaid
flowchart TD
Start(["开始局部自动布局"]) --> CheckCount{"选中节点数量≥2？"}
CheckCount --> |否| End(["结束"])
CheckCount --> |是| FilterEdges["过滤边：仅保留选中节点间连接"]
FilterEdges --> CalcBounds["计算原始包围盒"]
CalcBounds --> BuildGraph["构建ELK图结构仅选中节点"]
BuildGraph --> RunELK["调用 elk.layout(graph)"]
RunELK --> CalcOffset["计算偏移量：原始最小坐标-布局最小坐标"]
CalcOffset --> MapCoords["映射坐标：布局坐标+偏移量"]
MapCoords --> Replace["graphSlice.replace(更新节点, 所有边)"]
Replace --> End
```

### 对齐吸附（网格/边缘/中心）
- 对齐模式
  - Left：以最左侧节点的左边缘为基准对齐
  - Right：以最右侧节点的右边缘为基准对齐
  - Top：以最顶端节点的上边缘为基准对齐
  - Bottom：以最底端节点的下边缘为基准对齐
  - Center：以所有节点的水平中心线为基准进行水平居中对齐
  - Middle：以所有节点的垂直中心线为基准进行垂直居中对齐
- 吸附算法
  - 计算拖拽节点与其它节点的左右/上下关键点（left/centerX/right、top/centerY/bottom）
  - 在 X/Y 轴分别寻找最小距离，生成吸附偏移与参考线
  - 支持"仅可视范围内"过滤
- 参考线渲染
  - 将吸附参考线投影到画布坐标系，使用重复渐变线样式提示对齐位置

### 智能间距调整（整体均匀扩展/压缩）
- 基准点
  - 以目标节点集合中某一维度（横或纵）的最小位置为基准
- 缩放因子
  - 以每个节点与基准点的距离占"100像素"的比例，乘以 delta 得到偏移
- 应用
  - 更新节点位置并保存历史

```mermaid

Base --> Scale["对每个节点计算缩放因子 = 距离/100"]
Scale --> Offset["offset = scale * delta"]
Offset --> Move["更新节点位置"]
Move --> Save["保存历史记录"]
```

## 性能考虑
- 自动布局

  - 局部自动布局仅处理选中节点，显著减少计算量

  - 按比例缩放的偏移计算为 O(n)，适合大规模节点集
- 状态更新
  - replace 接口一次性批量更新，减少多次重绘

## 结论
MaaPipelineEditor 的布局体系以 ELK 分层布局为核心，辅以对齐吸附与智能间距调整，既保证了复杂工作流的自动化排版，又保留了手动微调的能力。新增的局部自动布局功能进一步增强了布局系统的灵活性，允许用户对选定节点进行智能定位而不影响其他节点。通过配置项与状态切片的协同，系统实现了稳定的布局体验与良好的可维护性。工具栏按钮的智能切换机制让用户能够根据场景选择合适的布局模式，提升了用户体验。

## 附录
- 复杂工作流布局策略
  - 先使用自动布局获得全局结构，再通过对齐与间距调整优化局部细节
  - 对于长链路，优先保证层间间距与方向一致性，避免边交叉过多
  - 使用 Middle 和 Center 对齐选项处理不同尺寸的节点组合
  - 局部自动布局适用于修复特定区域的布局问题，而无需重新计算整个流程图
- 性能优化建议
  - 控制节点数量与吸附范围，必要时开启"仅可视范围内吸附"
  - 合理设置 ELK 参数，避免过度优化导致计算时间过长
  - 使用 replace 接口进行批量更新，减少频繁小改动带来的重绘成本
  - 利用 Center 和 Middle 对齐选项减少手动微调次数
  - 局部自动布局仅处理选中节点，适合大规模工作流的精细化调整

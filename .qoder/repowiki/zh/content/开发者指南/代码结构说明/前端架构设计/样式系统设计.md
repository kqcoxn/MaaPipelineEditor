## 简介
本文件系统性阐述 MaaPipelineEditor 的样式系统架构设计，重点覆盖以下方面：
- Less 组织结构：全局样式、模块化样式、主题样式分层
- 命名规范与隔离：BEM 方法论、CSS Modules 使用、样式作用域隔离
- Ant Design 集成与定制：主题变量、组件覆盖、响应式设计
- 性能优化：样式分割、按需加载、缓存策略
- 调试与维护：实用指南、团队协作建议

**更新** 样式系统已完成重构，从单一目录结构迁移到模块化子目录结构，并新增探索面板专用样式模块，提升了系统的可维护性和扩展性。

## 项目结构
样式系统采用"入口聚合 + 层次化模块 + 子目录分类"的组织方式：
- 入口聚合：通过 index.less 汇总 base、flow、layout、panels 等基础样式
- 基础样式：base 目录提供全局工具类、Ant Design 覆盖、图标字体样式
- 流程图样式：flow 目录定义连接线、节点、边等视觉变量与动画
- 布局样式：layout 目录管理应用整体布局、头部、工具栏等样式
- 面板样式：panels 目录包含各个功能面板的专用样式，新增探索面板专用样式模块
- 主题样式：通过 ThemeContext 控制暗色模式，结合 DarkReader 实现自动适配

### 探索面板专用样式系统

#### 悬浮球(FAB)样式
- 基础样式：圆形悬浮球，使用渐变背景和阴影效果
- 交互状态：悬停放大、按下缩放、脉冲动画、激活状态、禁用状态
- 动画效果：1.5秒脉冲动画，平滑的变换过渡
- 位置定位：固定在右下角，z-index 层级较高确保可见性

#### 中控面板样式
- 布局结构：绝对定位面板，包含头部、内容区域、底部按钮
- 状态切换：通过 opacity 和 transform 实现淡入淡出和缩放效果
- 暗色模式适配：使用 :global(.dark) 伪类适配深色主题
- 响应式设计：基于百分比的宽度和居中定位

#### 幽灵节点样式
- 视觉效果：虚线边框(#1890ff)、半透明背景(rgba)、透明度控制
- 暗色模式支持：深色模式下调整背景透明度
- 操作按钮：浮动的操作按钮组，包含确认、取消等操作
- 位置定位：相对于节点的绝对定位，支持复杂的交互布局

#### 状态管理系统
- idle 状态：目标输入、起始节点选择、开始按钮
- predicting 状态：进度指示、取消按钮
- reviewing 状态：目标显示、步骤信息、执行和确认按钮
- executing 状态：执行进度指示
- completed 状态：完成提示和关闭按钮

### 主题系统与暗色模式
- 主题上下文：ThemeContext.tsx 提供 isDark、toggleTheme、setTheme 接口

## 性能考量
- 样式分割与按需加载
  - 将页面/组件样式拆分为独立 module.less，配合构建工具按需打包，减少初始 CSS 体积
  - 入口仅引入必要基础样式（base、flow、layout、panels），避免冗余
  - **新增** 探索面板样式按需加载，仅在探索功能使用时加载
- 缓存策略
  - 利用浏览器缓存与构建产物指纹化，确保样式更新后可被正确替换
  - DarkReader 注入的暗色样式为运行时注入，避免额外静态资源
  - **新增** CSS Modules 生成的类名具有唯一性，避免样式冲突
- 选择器复杂度与重绘
  - 优先使用类选择器，避免深层后代选择器导致的重绘
  - 使用 CSS 变量与工具类（如 ellipsis）降低重复定义
  - **新增** 探索面板使用 :global(.dark) 伪类，避免过度嵌套
- 动画与过渡
  - 合理使用 transition 与 transform，避免频繁触发回流
  - 控制点与边的动画采用轻量级属性，保持流畅体验
  - **新增** 悬浮球脉冲动画使用 transform 和 box-shadow，性能优化良好

**更新** 新的目录结构使得样式文件更加模块化，新增探索面板样式进一步提升了按需加载的效率。

## 故障排查指南
- 样式未生效
  - 检查 index.less 的引入顺序是否正确
  - 确认组件是否使用了正确的 CSS Modules 类名
  - **新增** 检查探索面板样式文件路径是否正确
- AntD 组件样式异常
  - 查看 antd.less 是否覆盖了目标组件类名
  - 注意 !important 的使用场景与副作用
- 暗色模式不生效
  - 确认 ThemeContext.tsx 中 useDarkMode 配置已更新
  - 检查 DarkReader 是否成功启用/禁用
  - **新增** 检查 :global(.dark) 伪类是否正确应用到探索面板样式
- 响应式问题
  - 检查媒体查询断点与组件布局是否匹配
  - 确保容器具备足够的高度/宽度以容纳内容
- 目录结构问题
  - 确认样式文件路径是否正确指向新的子目录结构
  - 检查 @import 语句中的相对路径是否匹配新的目录层级
  - **新增** 检查探索面板样式文件是否存在于正确的 panels 目录中

## 结论
MaaPipelineEditor 的样式系统通过"入口聚合 + 层次化模块 + 子目录分类 + 主题上下文"的架构实现了清晰的职责分离与良好的扩展性。新的目录结构将样式功能细分为 base、flow、layout、panels 四个主要类别，显著提升了系统的可维护性和模块化程度。新增的探索面板专用样式模块进一步完善了样式系统的功能覆盖，包含了完整的状态管理、动画效果和响应式设计。

Less 的变量与工具类提升了复用性，CSS Modules 确保了样式隔离，AntD 定制满足产品化需求，DarkReader 提供了便捷的主题能力。探索面板的幽灵节点视觉效果、悬浮球交互、中控面板等创新设计展现了样式的灵活性和实用性。

配合合理的性能策略与调试指南，可在保证开发效率的同时维持高质量的用户体验。新的样式系统架构为未来的功能扩展和维护提供了更好的基础。

**更新** 样式系统的重构不仅改善了代码组织结构，新增探索面板样式模块进一步增强了系统的功能完整性和用户体验。

## 附录
- 命名规范与最佳实践

  - 作用域：页面/组件样式统一使用 CSS Modules，类名由构建工具生成唯一标识
  - 变量：在 flow.less 中集中定义颜色与尺寸变量，避免硬编码
  - 覆盖：AntD 覆盖尽量局部化，使用 :global 时注意作用域边界
  - 响应式：在组件级样式中使用媒体查询，避免全局污染
  - 目录：按照功能分类组织样式文件，base/flow/layout/panels 四大类清晰划分
  - **新增** 探索面板：使用语义化命名，如 fab、panel、ghostNode 等

  - 主题相关改动统一通过 ThemeContext 管理，避免分散配置
  - 遵循新的目录结构约定，按功能分类存放样式文件
  - 更新样式时注意保持与其他模块的依赖关系

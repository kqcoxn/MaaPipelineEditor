## 引言
本测试策略文档面向前端 React 应用与后端 Go 服务的全栈测试，目标是建立覆盖单元测试、集成测试与端到端测试的分层策略，明确组件渲染、用户交互、状态管理、HTTP 接口、WebSocket 通信等关键测试场景，并提供 Mock 数据准备与使用方法、测试覆盖率要求以及在持续集成中执行测试的建议。同时涵盖性能测试、安全测试与兼容性测试的专项策略。

**更新** 新增着陆页（Landing）测试策略，采用基于静态服务器的 Playwright 测试方法，提供更稳定和可控的测试环境。

## 项目结构
本仓库包含四部分：
- 前端编辑器应用（React + TypeScript），位于 src 目录，使用 Vite 作为构建工具与 Vitest 作为测试框架。
- 本地桥接服务（LocalBridge）：Go 实现的本地服务，提供文件扫描、资源管理、MFW 协议处理、WebSocket 通信等能力。
- Extremer：基于 Wails 的桌面应用壳层，负责与本地服务交互并提供 UI。
- **着陆页（Landing）**：Astro 构建的静态网站，使用 Playwright 进行端到端测试，通过静态服务器脚本提供稳定的测试环境。

### 前端组件测试策略
- 组件渲染测试
  - 使用 @testing-library/react 与 happy-dom 环境，断言组件挂载、DOM 结构与可见性。
  - 对于 Flow 画布、节点列表、工具栏等关键区域进行快照与交互测试。
- 用户交互测试
  - 模拟点击、拖拽、输入等事件，验证节点上下文菜单、工具面板、字段面板的行为。
  - 使用屏幕读屏与无障碍断言确保可访问性。
- 状态管理测试
  - 针对 Zustand store（如 flow、debug、mfw、wsStore）编写单元测试，覆盖切片逻辑与派生状态。
  - 使用测试替身隔离外部依赖（如 WebSocket、HTTP）。
- 协议与网络测试
  - 对 DebugProtocol 的消息组装、结果展示逻辑进行单元测试。
  - 使用 Vitest 的 mock 功能模拟 WebSocket 发送与响应，验证错误分支与边界条件。
- JSON 辅助工具测试
  - 对 JsonHelper 的对象判断、字符串解析与序列化进行单元测试，覆盖异常输入与边界情况。

### 后端 Go 服务测试策略
- 单元测试
  - 对核心模块（如文件服务、资源服务、MFW 类型定义）编写单元测试，覆盖业务逻辑与错误处理。

### 着陆页测试策略
**更新** 新增着陆页测试策略，采用基于静态服务器的 Playwright 测试方法

- **静态服务器脚本**
  - run-playwright-with-static-server.cjs 提供本地静态服务器，监听 127.0.0.1:4321。
  - 自动检查 dist 目录存在性，确保构建产物可用。
  - 支持多种文件类型的 MIME 类型处理，包括 CSS、HTML、JavaScript、图片等。
  - 通过 child_process 启动 Playwright 测试，测试完成后自动关闭服务器。
- **Playwright 配置**
  - playwright.config.ts 使用 Astro 的 preview 命令启动 WebServer。
  - 配置单项目（chromium）和基本的测试选项。
  - 设置 baseURL 为 http://127.0.0.1:4321，确保测试与静态服务器同步。
  - 支持 CI 环境下的重试机制和报告器配置。
- **测试场景**
  - **导航测试**：验证英雄区域的 CTA 按钮指向正确目的地。
  - **功能标签测试**：验证功能切换标签的内容变化和键盘导航支持。
  - **移动端导航测试**：验证移动端菜单的打开/关闭和锚点导航。
  - **桌面导航测试**：验证桌面端生态链接和 GitHub 链接的正确性。
- **测试最佳实践**
  - 使用测试 ID（data-testid）进行元素选择，提高测试稳定性。
  - 验证 ARIA 属性确保可访问性。
  - 使用视口设置测试移动端适配。
  - 通过 URL 断言验证锚点导航功能。

### 端到端测试策略
- **传统端到端场景**
  - 从打开应用到完成一次"测试节点/测试识别"的完整链路，验证 UI、状态、协议与 WebSocket 的协同。
- **着陆页端到端场景**
  - **静态服务器端到端**：通过 run-playwright-with-static-server.cjs 启动本地静态服务器，提供稳定的测试环境。
  - **页面交互端到端**：验证着陆页的导航、功能展示、移动端适配等完整用户体验。
- **测试步骤**
  - 启动 LocalBridge 与前端（或 Extremer），准备测试资源与控制器连接。
  - 对于着陆页，执行静态服务器脚本启动 Astro 预览服务器，然后运行 Playwright 测试。
  - 执行节点上下文菜单中的测试命令，观察调试协议返回的结果与 UI 反馈。
  - 对着陆页执行导航、功能切换、移动端菜单等交互测试。
- **覆盖范围**
  - 包含错误场景（未连接、无控制器、资源路径为空）与成功场景（识别命中、动作执行）。
  - 包含着陆页的导航错误、功能标签切换失败、移动端菜单无法打开等场景。

## 故障排查指南
- 常见问题
  - 未连接 LocalBridge：前端提示"请先连接 LocalBridge"，需检查服务启动与网络可达性。
  - 无控制器：提示"请先连接控制器"，需确认设备连接与权限。
  - 资源路径为空：提示"请先配置资源路径"，需在调试面板中添加有效路径。
  - **着陆页测试问题**：
    - dist 目录缺失：静态服务器脚本会提示需要先运行构建命令。
    - 服务器启动失败：检查端口占用和权限问题。
    - 测试超时：增加 webServer.timeout 配置或优化构建时间。
- 错误处理
  - WebSocket 发送失败时，前端应停止调试状态并提示错误。
  - 协议层对识别命中、动作执行与节点整体进行分类统计与延迟测量。
  - **静态服务器错误处理**：脚本会捕获文件读取错误并返回 500 状态码。
- 调试建议
  - 启用 LocalBridge 日志级别，捕获协议消息与错误堆栈。
  - 使用浏览器开发者工具与网络面板检查 WebSocket 握手与消息往返。
  - **着陆页调试**：使用 Playwright 的 trace 功能捕获测试过程，检查静态服务器响应。

## 简介
MaaPipelineEditor（MPE）是一个前后端分离的可视化 Pipeline 编辑器，基于 React 19 + TypeScript + Vite 构建前端，使用 Go 语言实现本地桥接服务（LocalBridge），并通过 Wails 将 Go 后端打包为桌面应用（Extremer）。项目提供在线版本与本地一体化应用两种使用形态，支持文件管理、截图工具、流程调试、AI 辅助等功能。

## 项目结构
项目采用多模块组织方式：
- 前端模块：位于根目录 src，使用 Vite + React 19 + TypeScript 构建
- 后端模块：LocalBridge（Go），提供 WebSocket 服务与 MaaFramework 集成
- 打包模块：Extremer（Go + Wails），将前端静态资源嵌入并打包为桌面应用
- 文档站点：docsite（VitePress），用于文档发布
- 工具脚本：tools（安装脚本）

### 前端应用初始化与连接流程
前端在启动时初始化 WebSocket，根据运行环境（在线或 Wails）决定连接策略：
- 在线环境：读取 URL 参数或配置，自动连接指定端口。
- Wails 环境：等待后端事件桥接发送端口，再发起连接。

### Wails 打包器生命周期
Extremer 负责应用启动、配置加载、本地服务启动与事件桥接：
- 启动阶段：检测开发模式、加载/创建配置、分配端口、准备 LocalBridge 配置。
- DOM 就绪：等待 LocalBridge 就绪后，向前端发送端口与就绪事件。
- 关闭阶段：停止 LocalBridge 子进程。


### LocalBridge 服务核心流程
LocalBridge 提供文件扫描、资源扫描、MFW 初始化、协议路由与日志推送：
- 安全检查：对根目录进行安全评估，避免高风险扫描。
- 服务启动：启动文件服务、资源扫描服务、MFW 服务（可选）。
- 协议注册：注册文件、MFW、调试、资源等协议处理器。
- 日志推送：将日志广播给前端 WebSocket 客户端。

## 性能考虑
- 前端性能：使用 React.memo、Zustand 状态分区、按需加载组件与资源。
- 后端性能：文件与资源扫描限制深度与数量，MFW 初始化失败降级为文件管理功能。
- WebSocket：日志推送批量处理，连接状态监控与自动重连。

## 故障排除指南
- 本地服务无法启动：检查 LocalBridge 配置文件路径、MaaFramework 路径与资源路径，查看日志目录。
- WebSocket 连接失败：确认端口分配与防火墙设置，检查前端 URL 参数与配置。
- Wails 环境连接异常：确认 Extremer 已发送 bridge:port 事件，检查启动画面与窗口显示逻辑。
- 安装脚本问题：Windows 使用 PowerShell 或 CMD 脚本，Linux/macOS 使用 Bash 脚本，确保网络可达与权限正确。

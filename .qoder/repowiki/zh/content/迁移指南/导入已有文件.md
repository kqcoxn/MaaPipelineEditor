## 简介
本指南面向希望将“已有文件”导入到 MaaPipelineEditor 的用户与开发者，系统讲解导入系统的整体架构与工作流程，涵盖文件格式识别、版本检测、数据解析、配置拆分与合并、错误处理与恢复机制，并提供导入前准备、导入后验证与常见问题排查的实操建议。

## 项目结构
导入能力由前端 UI 控件触发，经解析器模块完成格式识别、版本检测、数据转换与验证，再写入可视化画布与文件状态管理。关键路径如下：
- 前端入口：工具栏导入按钮组件
- 核心解析：导入器、版本检测器、配置解析器、节点解析器、边链接器、配置拆分器
- 状态与持久化：文件状态存储、跨文件服务

## 核心组件
- 导入按钮组件：提供从剪贴板、文件导入 Pipeline 与配置的能力，并根据配置处理模式动态展示菜单项。
- 导入器：统一协调解析流程，包括键序保留、配置合并、废弃字段迁移、节点解析、边连接、布局与历史初始化。
- 版本检测器：识别节点的识别/动作字段版本（v1/v2），并进行类型标准化。
- 配置解析器：识别并提取配置键与标记对象，兼容新旧版本标记。
- 节点解析器：按协议版本将字段映射到节点数据结构，处理 v1 平铺参数与 v2 对象参数。
- 边链接器：解析 next/on_error 引用，支持外部节点/锚点/回跳等语义。
- 配置拆分器：将“混合配置+管道”拆分为“纯管道+分离配置”，或反向合并。
- 文件状态存储：负责导入后的内容写入、同步状态更新与保存确认。
- 跨文件服务：用于节点引用的文件级解析与前缀推断。

### 导入器（核心流程）
- 参数获取与空内容处理：支持从剪贴板读取或显式传入字符串；对空/全空格/null 进行兜底。
- 键序保留：使用 JSONC 解析器遍历对象属性，记录第一层键序，保证导入后顺序稳定。
- 配置合并：当存在外部 MPE 配置时，先合并再解析，确保位置、前缀等元信息生效。
- 配置解析：提取配置键与标记对象，兼容新旧版本标记。
- 废弃字段迁移：将 v5.1 的中断/子节点语义迁移到新的 JumpBack 语义，修正 next/on_error 引用。
- 节点解析：识别外部节点/锚点/普通节点，解析识别/动作/其他字段，处理位置与端点方向。
- 边连接：解析 next/on_error，支持字符串/对象两种引用形式，必要时创建外部/锚点节点。
- 布局与历史：若无位置信息则自动布局；初始化历史记录；更新文件名/前缀/节点顺序映射。
- 错误处理：捕获异常并弹窗提示，同时输出控制台错误日志。

### 版本检测器（协议版本识别）
- 识别规则：
  - recognition/action 字段为对象且包含 type → v2
  - recognition/action 字段为字符串 → v1
  - 若节点包含 v1 特征键（来自字段模式表）但未显式 type → v1
  - 默认返回 v2
- 类型标准化：对识别/动作类型进行大小写归一化与合法性校验，非法类型抛错。

## 性能考虑
- 键序保留：通过解析器遍历对象属性记录第一层键序，避免重复解析带来的开销。
- 懒创建外部节点：仅在引用缺失时创建外部/锚点节点，减少不必要的节点创建。
- 自动布局：仅在无位置信息时触发，避免重复布局计算。
- 配置合并：在导入器内一次性完成，减少多次序列化/反序列化成本。

## 结论
导入系统通过“UI 触发 + 解析器管线 + 状态存储”的架构，实现了对多种输入源与协议版本的兼容处理。版本检测器与节点解析器确保不同协议版本的平滑迁移，配置拆分器支持分离存储模式下的灵活导入。结合完善的错误处理与恢复机制，用户可以高效地将既有文件安全地导入到编辑器中。

## 附录

### 导入前准备清单
- 准备好要导入的 Pipeline/配置文件（支持 JSON/JSONC）。
- 若采用分离存储模式，准备好对应的 .mpe.json 配置文件。
- 确认文件编码为 UTF-8，避免隐藏字符影响解析。
- 如需保留节点位置，请在导出时启用位置标记。

### 导入流程与验证步骤
- 步骤
  - 在工具栏点击“导入”，选择“从文件导入 Pipeline”或“从粘贴板导入 Pipeline”。
  - 若采用分离存储模式，先“从文件导入配置”，再“从文件导入 Pipeline”。
  - 导入完成后，检查画布节点、边连接与文件名/前缀是否正确。
- 验证
  - 使用“自动布局”确保节点分布合理。
  - 手动运行一次流程，确认 next/on_error 连接与外部节点/锚点行为符合预期。
  - 导出验证：切换到分离存储模式，导出配置与管道，确认二者可独立加载。

### 常见问题与对策
- 问题：导入后节点顺序错乱
  - 对策：确认导入时是否正确保留了键序；必要时重新导出并导入。
- 问题：识别/动作类型报错
  - 对策：使用标准化类型，避免大小写差异；检查类型是否在允许范围内。
- 问题：next/on_error 无法连接
  - 对策：检查引用格式与前缀；确认目标节点是否存在或是否应创建外部节点。

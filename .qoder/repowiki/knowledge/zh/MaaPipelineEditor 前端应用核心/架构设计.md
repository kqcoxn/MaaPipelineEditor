- **启动编排**：在 `src/main.tsx` 中统一初始化 WebSocket 服务 (`initializeWebSocket`) 与开发控制台，随后挂载根组件。
- **状态协调**：`App.tsx` 作为顶层容器，负责协调多个独立 Store（如 `flowStore`, `fileStore`, `embedStore`）的生命周期，并通过 Context 提供全局主题支持。
- **通信桥接**：采用双模通信架构。内部通过 `LocalWebSocketServer` 封装 WebSocket 连接并注册多业务协议（File, MFW, Debug 等）；外部通过 `embedBridge` 实现基于 `postMessage` 的 iframe 嵌入握手与能力协商。
- **布局互斥**：利用自定义 Hooks (`usePanelOccupancy`) 与 Store 状态实现画布周边面板（如属性栏、日志区）的互斥占位与动态显隐，确保复杂编辑场景下的 UI 秩序。

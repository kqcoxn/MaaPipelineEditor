- **多态构建编排**：根目录 `package.json` 作为任务调度中心，通过 `yarn --cwd` 委托子模块（Landing, LocalBridge, docsite）执行独立构建，并利用 Vite `mode` 区分 Web 版与桌面版（Extremer）的资源路径。
- **跨平台发布流水线**：`.github/workflows/release.yaml` 定义了复杂的依赖编排，并行编译 Go 语言后端（LocalBridge/Extremer）与前端资源，自动拉取 MaaFramework 运行时，最终聚合为多平台安装包。
- **协议化通信边界**：确立了以 WebSocket 为核心的本地通信契约，连接 React 前端与 Go 后端，并通过 `Iframe/` 目录定义嵌入宿主环境的标准化交互接口。

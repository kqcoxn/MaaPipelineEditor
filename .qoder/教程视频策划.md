# MaaPipelineEditor (MPE) 入门视频策划案

## 制作流程

配音 → 根据配音录制画面 → 剪辑

---

## 完整配音稿

---

如果你写过 MaaFramework 的 Pipeline，应该体会过在几千行 JSON 里找节点的痛苦。改一个参数，要翻半天；加一个新步骤，要小心翼翼地改 next 引用，生怕写错一个地方整个流程就崩了。

MaaPipelineEditor，简称 MPE，就是来解决这个问题的。它是一个可视化的 Pipeline 编辑器，打开网页就能用，不用安装任何东西。接下来几分钟，我带你从零开始，搭建第一个自动化流程。

---

进入编辑器后，先看下界面布局。左边是工具栏，可以添加节点，也能打开设置。中间是画布，所有操作都在这里进行。右边是字段面板，选中节点的时候，会显示这个节点可以配置的参数。

左上角有个设置按钮，可以切换主题，亮色的暗色的都有，看个人习惯。

---

我们先创建第一个节点。在画布空白处右键，会弹出模板面板。这里有很多预设模板，都是常用的节点组合。我们先选一个最简单的空节点。

节点创建之后，右侧的字段面板会显示这个节点的配置。每个节点都有一个名字，默认叫"新建节点1"，没什么意义，我们把它改成"开始示例"。改完之后，节点上显示的名称也会同步更新。

---

接下来创建一个能识别文字的节点。保持刚才的节点选中状态，再次右键添加，新节点会自动连到选中节点后面。这次我们选"文字识别"模板。

选中这个新节点，看右侧的字段面板。OCR 相关的字段已经自动配置好了，我们只需要在 expected 字段里输入要识别的文字，比如"显示"。这样配置之后，MaaFramework 运行的时候，会在屏幕上找这段文字，找到了就执行下一步。

鼠标移到字段名上，会显示这个字段的说明，不确定的时候可以看看。

---

我们继续添加节点。现在创建一个滑动操作的节点。这次没有现成的模板，我们从空节点开始，然后把动作类型改成 Swipe。

修改之后，字段面板会自动筛选 Swipe 动作可以配置的参数。我们点击添加字段，把 begin 和 end 加上，配置滑动的起点和终点坐标。

然后再加两个文字识别节点，一个识别"存储"，一个识别"无障碍"。

画布上的节点可以直接拖拽调整位置。按住 Shift 可以框选多个节点，右下角还有排版工具，选中多个节点之后可以自动对齐。

---

现在来说一下节点之间的连线。连线表示执行顺序。每个节点右侧有两个出口：上面是 next，正常执行的时候走这里；下面是 on_error，出错的时候走这里。左侧是入口，可以接收多个来源的连接。

直接拖拽端点就能创建连接。线上的数字表示执行顺序，也就是 next 数组里的索引。

如果要删除连接，先选中连线，然后按 Delete 或者 Backspace 就行。

---

流程搭建完成之后，点击右侧的 Pipeline JSON 面板，可以预览生成的 JSON 代码。点击导出按钮，可以复制到剪贴板，或者下载成文件。这就是 MPE 生成的 Pipeline JSON，可以直接给 MaaFramework 使用。

反过来，如果你已经有 JSON 文件，也可以直接粘贴或者拖拽导入，MPE 会自动解析，生成对应的流程图。

---

这就是 MPE 的基本用法：拖拽创建节点，右侧配置字段，连线确定执行顺序，最后导出 JSON。本地服务、调试工具、AI 辅助这些进阶功能，可以查看官方文档了解。文档链接放在视频简介。

## 三、制作规范

### 3.1 视频规格

- **分辨率**：1920x1080（1080P）
- **帧率**：30fps
- **音频**：清晰人声，背景音乐音量低于人声
- **字幕**：关键操作步骤需配字幕

### 3.2 界面展示规范

- 使用 MPE 默认主题或暗色主题
- 重要操作配合鼠标高亮或点击特效
- 关键配置项使用放大或标注

### 3.3 讲解风格

- 语速适中，重点内容适当放慢
- 专业术语首次出现时解释
- 鼓励观众动手跟着做

### 3.4 配套材料

每集视频应提供：

- 视频简介（含时间戳）
- 演示用的示例文件（如有）
- 相关文档链接
- 下一集预告

## 五、附录

### 5.1 常用术语对照

| 术语 | 英文                          | 说明                      |
| ---- | ----------------------------- | ------------------------- |
| 节点 | Node                          | Pipeline 中的单个任务单元 |
| 任务 | Task                          | 由多个节点组成的完整流程  |
| 入口 | Entry                         | 任务的第一个节点          |
| 识别 | Recognition                   | 图像/文字识别操作         |
| 动作 | Action                        | 点击、滑动等执行操作      |
| ROI  | Region of Interest            | 感兴趣区域，限定识别范围  |
| OCR  | Optical Character Recognition | 光学字符识别              |

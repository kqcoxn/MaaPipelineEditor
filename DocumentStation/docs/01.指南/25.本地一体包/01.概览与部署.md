MaaPipelineEditor Desktop 是基于 Tauri 2 的桌面宿主。它复用同一套 Editor，并负责 Python 环境、LocalBridge 子进程、工作区选择、托盘、快捷键和签名更新；业务逻辑仍全部位于 Python LocalBridge。

## 支持平台

- Windows x64：NSIS 安装包
- macOS Apple Silicon：DMG
- Linux x64：AppImage

安装包从 [GitHub Releases](https://github.com/kqcoxn/MaaPipelineEditor/releases) 获取。

## Python 前置条件

Desktop 不内置 Python，也不会安装、升级或修改系统 Python。系统需要存在同架构 CPython 3.11-3.14。缺失时应用会进入修复页并提供当前平台的安装指引。

首次启动需要联网。Desktop 会在应用数据目录创建版本化私有 venv，并安装与 Desktop 严格同版本的 `MaaPipelineEditor-LocalBridge`。环境通过 `mpelb doctor --json` 后才会原子切换；之后可离线启动，上一版本环境会保留用于应用回滚。

## 工作区与本地数据

首次启动会创建默认工作区，也可以在桌面设置中选择其他目录。Tauri 只允许 WebView 打开工作区和应用数据目录内的路径，不提供通用 shell 或全盘文件权限。

LocalBridge 使用随机端口和随机 token。ready 信息通过受限 stdio NDJSON 通道交给 Tauri，再通过 bootstrap command 交给 Editor，普通日志不会混入控制通道。

## 后台驻留

默认关闭窗口即退出应用。只有显式开启“后台驻留”后，关闭窗口才改为隐藏，并启用托盘、开机启动和全局快捷键。

默认快捷键：

- 显示/隐藏：`CommandOrControl+Shift+Alt+M`
- 停止调试：`CommandOrControl+Shift+Alt+S`

快捷键可以修改；注册冲突时保留原值并显示错误。Linux Wayland 或无托盘环境会显示能力降级状态，但不阻止核心编辑和调试。

## 更新

Desktop 只安装签名更新，并要求用户明确确认。安装前会先准备目标版本 Python venv 并通过 doctor，随后安装 Desktop 更新并重启。若目标 Python 环境无法准备，Desktop 不会进入更新安装阶段。

## 故障处理

LocalBridge 异常退出后会按 1、2、4 秒退避自动重启，最多三次；之后进入可重试的修复状态。宿主退出时会先请求五秒优雅停止，再通过 Windows Job Object 或 Unix 进程组终止完整子进程树。

### 首次启动无法安装

确认 Python 架构与 Desktop 一致，并确认可以访问 PyPI。修复后在页面中重新探测。

### 更新检查失败

确认网络可以访问 GitHub Release。手动下载安装包不会删除当前私有 venv 和工作区设置。

### OCR 不可用

MaaFw wheel 包含运行库，但业务 OCR 模型由资源项目提供。请在 LocalBridge 配置中设置对应的 `maafw.resource_dir`。

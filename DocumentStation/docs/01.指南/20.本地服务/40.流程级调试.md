流程级调试用于在 MaaPipelineEditor 中直接运行当前 Pipeline 图，观察节点执行路径、识别结果、动作详情、截图产物和运行诊断。调试能力由 LocalBridge 调用 MaaFramework 提供，编辑器负责把运行事件映射回画布节点。

调试入口集中在 **调试工作台** 中。它不是一个需要开启或关闭的“调试模式”，也不提供断点、暂停、继续、单步等传统调试器语义；需要定位某个节点时，可以从指定节点重新发起一次新的运行。

## 前置条件

使用调试前，请确认：

1. 已启动并连接 LocalBridge。
2. 已初始化 MaaFramework，并连接 ADB 或 Win32 控制器。
3. 已配置可加载 Pipeline、image 等资源的资源路径。
4. 当前图中存在可映射到运行时名称的 Pipeline 节点。

::: tip
纯 Web 端无法访问本地 MaaFramework runtime，因此不提供流程级调试。Web + LocalBridge 和本地一体包均可使用。
:::

## 打开调试工作台

点击全局工具区的调试按钮即可打开调试工作台。工作台左侧按使用频率组织：

- **调试配置**：集中管理资源路径、控制器、代理（Agent）和产物策略（Artifact Policy）。
- **中控台**：启动、停止、选择入口节点、配置 Pipeline Override，并查看当前/最新运行摘要。
- **节点线**：按节点聚合执行记录，查看执行路径、事件详情和产物（截图、识别图、动作详情等）。
- **资源体检**：在运行前自动检测资源路径、资源加载结果和流程图结构。
- **调试日志**：查看本次调试运行生成的 `maafw.log` 末尾内容，并快速打开日志所在文件夹。

LocalBridge 未连接时也可以打开工作台；连接后会读取调试能力清单（Capability Manifest），界面会按真实后端能力启用按钮。

## 调试配置

### 资源路径

资源路径用于加载 Pipeline、image 和其他运行资源。可以手动输入，也可以使用 LocalBridge 扫描到的资源包（Resource Bundle）。

路径通常应指向包含 `pipeline`、`image` 等子目录的资源根目录。多个路径会按顺序加载，后加载的同名资源覆盖先加载的资源。

### 控制器

实时运行会使用当前已连接的 MaaFramework 控制器（Controller）。控制器的连接与切换在本地服务中完成，调试工作台仅显示当前控制器状态。

### MFW Agent

Agent（MFW Agent）是 MaaFramework 提供的进程外扩展机制：自定义识别（Custom Recognition）与自定义动作（Custom Action）运行在一个独立进程（AgentServer）中，通过 IPC 或 TCP 与调试端的 AgentClient 通信。当 Pipeline 执行到自定义节点时，AgentClient 会把请求转发给 AgentServer，并在执行完成后取回结果。关于 Agent 的概念与编写方式，详见 MaaFramework 应用指南的 Custom & Agent 章节。

Agent 为可选能力，只有当被调试的资源用到了自定义识别 / 动作节点时才需要配置。若你的 Pipeline 不含自定义节点，可以跳过本节。

#### 与完整 UI 的区别（重要）

在 MFAAvalonia 等完整 UI 中，UI 会读取 `interface.json` 的 `agent.child_exec` / `agent.child_args`，自动拉起 Agent 子进程并用随机标识符与之连接，使用者无感知。

而 MPE 的调试工作台**不会自动拉起 Agent 进程**。它只负责创建并连接 AgentClient，因此你需要：

1. 先**手动启动**外部的 AgentServer 进程，自定一个固定的标识符（或 TCP 端口）。
2. 在 MPE 中填写**相同**的标识符 / 端口，再发起连接。

两端的标识符 / 端口必须一致，连接才能成功。

#### 第一步：手动启动 Agent

以一个典型的 Python Agent 项目为例：例如 agent 的入口文件位于 `~/agent/main.py`，其入口通常把命令行最后一个参数作为 socket 标识符传给 `AgentServer.start_up`：

```python
# agent/main.py（节选）
socket_id = sys.argv[-1]
print(socket_id)  # 这里输出的内容就是 agent 标识符
AgentServer.start_up(socket_id)
AgentServer.join()
AgentServer.shut_down()
```

因此手动启动时，只需在末尾附上一个你自定的标识符即可。在项目根目录下执行：

```bash
# 标识符（Identifier）模式：mpe-agent 即为自定的标识符，可任意取名但需便于辨识
python agent/main.py mpe-agent
```

若项目内置了便携式 Python（如 `interface.json` 中 `child_exec` 指向 `./python/python.exe`），请改用对应解释器，并保持参数一致：

```bash
./python/python.exe -u ./agent/main.py mpe-agent
```

启动后保持该终端运行，控制台通常会输出等待连接的提示。注意：

- AgentServer 所使用的 maafw 库版本（Python / Go 的 maafw 绑定）应与 MPE 所依赖的 MaaFramework 版本一致，否则可能连接失败或行为异常。
- Agent 进程需要能加载与 MPE 调试相同的资源，因此请在配置好资源路径后再连接。

#### 第二步：在 MPE 中配置并连接

打开调试工作台 → 调试配置 → 代理（Agent），点击「添加代理（Agent）」，然后：

| 配置项                | 说明                                                                                            |
| --------------------- | ----------------------------------------------------------------------------------------------- |
| 启用开关              | 打开后该 Agent 才会在运行时参与连接。                                                           |
| 传输方式（Transport） | 选择「标识符（Identifier）」或「TCP」，需与启动 Agent 时使用的方式一致。                        |
| 标识符 / TCP 端口     | 标识符模式填写与启动命令末尾完全相同的字符串（如 `mpe-agent`）；TCP 模式填写 Agent 监听的端口。 |
| 超时（ms）            | 单次请求的超时时间，默认 2000ms，复杂识别 / 动作可适当调大。                                    |
| 必需                  | 勾选后，该 Agent 连接失败会阻断运行；取消勾选则失败时仅告警、继续运行。                         |

填写完成后点击「测试连接」。测试只验证外部已启动的 Agent 是否可达，**测试通过后会立即断开**，正式运行时会按当前配置重新连接。运行时还会把 custom recognition / action 的执行情况写入追踪事件。

::: tip
若使用 TCP 模式，启动 Agent 时同样需要让 AgentServer 监听对应端口（具体方式取决于 Agent 项目的实现，标识符模式更常用也更简单，推荐优先使用）。
:::

#### 连接失败排查

测试连接失败时，常见原因包括：

- **未启动 Agent**：MPE 不会自动拉起 Agent，需确认对应进程仍在运行。
- **标识符 / 端口不一致**：两端必须完全相同（区分大小写）。
- **版本不一致**：Agent 的 maafw 库版本与 MPE 依赖的 MaaFramework 版本不匹配（可通过替换 MPE 填写的 Lib 目录对齐）。
- **资源未加载**：连接前需先成功配置并加载资源路径。
- **必需项阻断**：若勾选了「必需」，连接失败会直接中断运行，可临时取消勾选以便先排查其他环节。

### 产物策略

产物（Artifact）是运行过程中按需读取的大对象。当前支持的产物类型包括：

- 原始图（Raw Image）
- 绘制图（Draw Image）
- 动作详情（Action Detail）

普通追踪事件只保存产物引用，以避免把大图直接塞进事件流造成卡顿或 OOM。

## 运行方式

### 完整运行

从配置档入口（Profile Entry）启动完整 Pipeline。适合验证整条流程是否能按预期执行。

### 从选中节点运行

以当前选中节点作为入口，继续沿 `next` / `on_error` 执行后续节点。适合复查某个中段节点之后的流程。

### 单节点运行

只对选中节点执行一次识别与动作组合，不继续运行后续节点。适合快速验证单个节点的完整配置。

### 仅识别

只测试选中节点的识别逻辑，不执行动作。适合调试 OCR、模板匹配、颜色匹配等识别参数。

### 仅动作

跳过识别，直接执行选中节点的动作（Action）。此模式会弹出确认，因为它可能直接点击、滑动或输入。



## 追踪与产物

调试运行会生成追加式追踪（Trace）。每个事件包含序号、时间、来源、事件类型、阶段、状态、节点映射和产物引用。

节点线面板按节点聚合这些追踪事件，并支持按事件类型、状态、是否含产物等方式筛选。点击某条执行记录后，在详情区中可以向 LocalBridge 请求加载对应的图片或 JSON 产物。

## Pipeline Override（运行时覆盖）

Pipeline Override 允许您在调试运行时临时覆盖节点的配置，无需修改原始 Pipeline 文件。适合快速验证参数调整、测试不同识别策略或跳过特定节点。

### 使用方式

在调试工作台的中控台面板中，展开 **Pipeline Override** 编辑区，输入覆盖配置 JSON。

### 格式说明

覆盖配置以节点的运行时名称（RuntimeName）为键，值为需要覆盖的字段：

```json
{
  "节点名称": {
    "recognition": "OCR",
    "action": "Click",
    "roi": [100, 200, 300, 400]
  },
  "另一个节点": {
    "next": ["跳转目标"]
  }
}
```

### 支持的覆盖类型

- **OverridePipeline**：覆盖节点的完整配置（识别、动作、参数等）
- **OverrideNext**：覆盖节点的后继节点列表
- **OverrideImage**：覆盖节点使用的图像数据

### 使用场景

- 临时修改识别参数（如 ROI、阈值）进行快速调参
- 跳过某些节点（覆盖 next 指向后续节点）
- 测试不同的动作配置而不修改源文件
- 调试时强制使用特定图片进行识别

::: tip
Override 配置仅在当前调试会话中生效，不会修改原始 Pipeline 文件。支持 v1 和 v2 协议格式，系统会自动进行版本适配。
:::

## 辅助面板

### 调试日志面板

调试日志面板用于查看当前调试会话在 LocalBridge 端生成的 `maafw.log` 日志文件。它适合在运行失败后快速定位 MaaFramework 层的底层错误、警告和运行轨迹，而无需离开编辑器去文件系统中翻找日志。

**功能特点**：

- **自动读取**：面板首次打开且已连接 LocalBridge 时，会自动加载 `maafw.log` 的末尾内容。
- **刷新**：点击「刷新」可重新读取日志，查看最新的运行记录。
- **打开所在文件夹**：点击「打开所在文件夹」可直接在系统文件管理器中定位到 `maafw.log`，若文件已存在还会自动选中该文件。
- **大文件截断**：当 `maafw.log` 超过约 256KB 时，仅展示末尾片段，并标注「仅显示末尾片段」，避免前端卡顿。
- **状态提示**：展示文件大小、最后修改时间，以及当前读取状态（加载中、成功、失败等）。
- **自动滚动到底部**：加载新内容后，日志视图会自动滚动到最下方，方便查看最新记录。

**使用前提**：

- 已连接 LocalBridge。
- 至少执行过一次调试任务，使 `maafw.log` 已经生成。若文件不存在，面板会提示「maafw.log 不存在，执行一次调试任务后再试」。

日志文件位置由 LocalBridge 的运行模式和 `log.dir` 配置决定，详见 [进阶配置](/guide/server/advance)。

### 资源体检面板

资源体检面板在调试运行前自动检测资源配置的完整性和正确性，提前发现潜在问题。检查分为三个阶段：

#### 资源路径解析

检查配置的资源路径是否存在、是否可读。路径解析失败时直接报错并给出修复建议。

#### 资源加载

对已解析到的资源目录执行实际加载，验证 Pipeline JSON 文件是否能被 MaaFramework 正确读取。加载失败时列出具体的文件名与失败原因。

#### 流程图校验

在不运行任务的情况下，对编辑器中当前流程图的结构做静态校验，提前暴露连线和节点层面的问题。具体检查项包括：

- **节点 runtimeName 缺失**：每个节点必须有唯一的 runtimeName
- **节点重复**：同一 fileId + nodeId 不能出现多次
- **runtimeName 冲突**：不同节点的 runtimeName 不能相同
- **边起点/终点缺失**：每条连线的两端必须填写 runtimeName
- **边指向不存在的节点**：连线引用的 runtimeName 必须在图中实际存在
- **调试目标缺失**：必须选定一个调试入口节点
- **调试目标信息不完整**：入口节点的 fileId、nodeId、runtimeName 必须齐全

校验全部通过时显示"流程图校验通过"；存在错误时逐条列出问题及修复建议。

**诊断级别**：

- **Error**：严重问题，可能导致运行失败
- **Warning**：潜在风险，建议修复
- **Info**：提示信息，供参考

**功能特点**：

- 按类别分组展示诊断结果
- 提供修复建议
- 支持从诊断结果快速跳转到相关文件或节点

## 停止与日志

点击“停止”会请求 LocalBridge 停止当前任务，并释放运行时资源。停止后，当前会话内已经收到的节点线记录、诊断事件和产物引用仍可查看。

长期排查请以 MaaFramework 自动生成的 `maa.log` 为准。MPE 不维护独立的长期调试历史库。

## 常见问题

### 提示缺少资源路径

请在调试配置中选择资源包，或手动添加包含 `pipeline` / `image` 的资源根目录。也可以刷新 LocalBridge 的资源包列表后再试。

### 提示控制器不可用

请先在本地服务中连接 ADB 设备或 Win32 窗口，再回到调试工作台启动运行。

### 识别图或详情没有显示

确认产物策略中启用了对应类型，例如绘制图（Draw Image）或动作详情（Action Detail）。普通追踪事件只显示引用，需要在节点线详情中点击加载。

### 回放 / 录制控制器不可用

当前 MaaFramework Go 绑定尚未提供可维护的 Replay / Record controller 创建能力。MPE 会在能力清单中将其标记为不可用，而不是显示不可执行的假按钮。

## 使用建议

- 先用“仅识别”确认识别参数，再用“单节点运行”验证识别与动作组合。
- 中段流程问题优先使用“从选中节点运行”，避免重复跑完整流程。
- 复杂识别问题优先保存截图，再离线分析。
- 遇到运行失败时，同时查看诊断、节点线详情、产物详情和 `maafw.log`。

## 配置文件

默认配置文件位于平台用户配置目录下的 `MaaPipelineEditor/LocalBridge/config.json`。实际路径可通过以下命令查看：

```bash
mpelb config path
```

LocalBridge 不迁移旧 Go 版配置；Pydantic 会拒绝未知字段。可用配置结构如下：

```json
{
  "server": {
    "host": "127.0.0.1",
    "port": 9066,
    "allowed_origins": ["https://mpe.codax.site"]
  },
  "file": {
    "root": ".",
    "exclude": [".git", ".venv", "node_modules"],
    "extensions": [".json", ".jsonc"],
    "max_depth": 0,
    "max_files": 10000
  },
  "log": {
    "level": "INFO",
    "dir": "<平台用户日志目录>",
    "push_to_client": true
  },
  "maafw": {
    "enabled": true,
    "resource_dir": "",
    "debug_mode": true
  }
}
```

## 配置说明

- `server.host`：只允许 loopback，不能暴露到局域网或公网。
- `server.port`：`0` 表示使用临时端口，Desktop 固定使用该方式。
- `server.allowed_origins`：允许建立连接和访问 artifact 的 Editor Origin。
- `file.root`：唯一可写工作区根目录。
- `file.exclude`：扫描时忽略的目录名。
- `file.extensions`：作为 Pipeline 扫描的文件扩展名。
- `file.max_depth`：`0` 表示不限制递归深度。
- `file.max_files`：单次工作区快照的文件上限。
- `log.dir`：LocalBridge 与 `maafw.log` 的日志目录。
- `log.push_to_client`：是否向 Editor 推送 LocalBridge 日志事件。
- `maafw.enabled`：是否启用 MaaFramework 相关能力。
- `maafw.resource_dir`：OCR 等业务资源根目录，不是 MaaFw native library 目录。
- `maafw.debug_mode`：是否启用 MaaFw 调试日志。

`MaaFw` Python wheel 自带 native library，因此没有 `maafw.lib_dir` 配置。

## 安全边界

LocalBridge 只允许绑定 loopback 地址，WebSocket 与 artifact 接口均检查 Editor Origin。文件路径会经过 realpath 与工作区包含关系校验，拒绝 `..`、符号链接逃逸和大小写绕过。

自部署 Editor 必须显式配置 Origin：

```bash
mpelb config set server.allowed_origins '["https://editor.example.com"]'
```

## 限制

- WebSocket 单帧上限为 8 MiB。
- 单个 artifact 上限为 64 MiB。
- 每连接最多同时处理 64 个 RPC 请求。
- 请求支持超时、取消和断线后的调试事件续传。

## 日志与事件数据库

普通日志位于 `log.dir`。调试事件使用 SQLite WAL 持久化，断线重连按 `afterSeq` 续传；Session 过期或销毁时会停止任务、释放 MaaFw handle 并清理其 artifact。

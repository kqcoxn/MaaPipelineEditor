# LocalBridge 指令汇总

包名为 `MaaPipelineEditor-LocalBridge`，命令名为 `mpelb`。

## 服务

```bash
mpelb serve [options]
```

| 参数 | 说明 |
| --- | --- |
| `--host <host>` | 监听地址，只接受 `127.0.0.1` 或 `localhost` |
| `--port <port>` | 监听端口，使用 `0` 时由系统分配 |
| `--root <path>` | Pipeline 工作区根目录 |
| `--allow-origin <origin>` | 追加允许的 Editor Origin，可重复指定 |
| `--config <path>` | 使用指定配置文件 |
| `--editor-url <url>` | 指定自动打开的 Editor 基础地址 |
| `--control-stdio` | 启用 Desktop 使用的 NDJSON 控制通道 |
| `--no-open` | 不自动打开在线 Editor |

示例：

```bash
mpelb serve --root ./Assets
mpelb serve --root ./Assets --port 0 --no-open
mpelb serve --allow-origin https://editor.example.com
```

## 环境检查与版本

```bash
mpelb doctor
mpelb doctor --json
mpelb version
```

`doctor --json` 适用于 CI、Desktop 私有 venv 验证和故障排查。

## 配置管理

```bash
mpelb config path
mpelb config show
mpelb config get <点路径键>
mpelb config set <点路径键> <JSON值或字符串>
```

示例：

```bash
mpelb config get file.root
mpelb config set file.root '"D:/Project/Assets"'
mpelb config set server.port 9066
mpelb config set maafw.debug_mode true
mpelb config set server.allowed_origins '["https://mpe.codax.site","https://editor.example.com"]'
```

字符串若也符合 JSON 语法，需要保留 JSON 引号。无法解析为 JSON 的输入会按普通字符串保存。

使用自定义配置文件：

```bash
mpelb config --config ./localbridge.json show
mpelb serve --config ./localbridge.json
```

## Desktop 控制通道

`--control-stdio` 只供 Tauri 宿主使用。stdout 第一行是 `ready` 消息，后续只接受 `health`、`stop_active_run` 和 `shutdown`；普通日志写入 stderr 和日志文件。不要把该模式用于通用 RPC。

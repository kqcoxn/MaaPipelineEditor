## 部署模式

MaaPipelineEditor 提供三种使用方式：

- **纯 Web**：只使用浏览器，适合查看、编辑和分享。
- **Web + LocalBridge**：手动运行 Python 后端，适合开发、多项目和自部署。
- **Desktop**：Tauri 2 桌面宿主管理私有 Python venv 与 LocalBridge，适合日常使用。

| 特性 | 纯 Web | Web + LocalBridge | Desktop |
| --- | --- | --- | --- |
| 可视化编辑与布局 | ✅ | ✅ | ✅ |
| Pipeline v1/v2 语法与导入 | ✅ | ✅ | ✅ |
| 分享、图片导出 | ✅ | ✅ | ✅ |
| 本地文件扫描、保存与监听 | ❌ | ✅ | ✅ |
| 跨文件导航与搜索 | ⚠️ 仅已打开文件 | ✅ | ✅ |
| ADB/桌面控制器与截图 | ❌ | ✅ | ✅ |
| OCR、模板匹配与资源预览 | ❌ | ✅ | ✅ |
| 流程与节点级调试 | ❌ | ✅ | ✅ |
| 事件续传、追踪与本地 Replay | ❌ | ✅ | ✅ |
| 工作区原生目录选择 | ❌ | ❌ | ✅ |
| 后台驻留、托盘与全局快捷键 | ❌ | ❌ | ✅，按平台降级 |
| 签名自动更新 | Web 自动部署 | 通过 PyPI 手动升级 | ✅，用户确认后安装 |

## 纯 Web

无需本地环境，可直接使用 [stable](https://mpe.codax.site/stable) 或 [preview](https://kqcoxn.github.io/MaaPipelineEditor/)。浏览器无法直接访问 MaaFramework native runtime 和完整本地文件系统，因此不提供设备、原生识别和调试能力。

## Web + LocalBridge

LocalBridge 是基于 Python、FastAPI 和官方 `MaaFw` Binding 的唯一业务后端，提供工作区文件、资源、MaaFramework、调试、工具和模型 API 代理能力。它只监听 loopback、校验 Editor Origin，并用 `/v2/ws` 提供协议 `2.3.0`。

该模式需要 CPython 3.11-3.14。推荐通过 `uv tool install MaaPipelineEditor-LocalBridge` 安装，然后在项目目录运行 `mpelb serve --root ./Assets`。详情见 [本地服务部署](/guide/server/deploy)。

## Desktop

Desktop 基于 Tauri 2，不复制 LocalBridge 业务逻辑。Rust 负责探测同架构 CPython、创建版本化私有 venv、启动和监督 Python 子进程，以及提供目录选择、托盘、快捷键和 updater 等桌面能力。

Desktop 不内置或修改系统 Python。首次启动需要联网安装与 Desktop 严格同版本的 LocalBridge；私有 venv 就绪后可离线使用。详情见 [Desktop 部署](/guide/desktop/deploy)。

## 如何选择

- 临时查看、学习或审阅：纯 Web。
- 资源开发、自部署或需要同时维护多个项目：Web + LocalBridge。
- 希望由应用管理后端生命周期、工作区与更新：Desktop。

三种模式使用相同的 Pipeline 文件。LocalBridge v2 不迁移旧版后端或桌面端配置，也不提供 v1 LocalBridge 协议兼容层。

## Node结构体详解

`Node`结构体是流水线配置的核心，定义了任务流中的单个节点。每个节点包含一系列字段，用于控制其行为和与其他节点的交互。

- **Name**: 节点的名称，作为其在流水线中的唯一标识符。
- **Anchor**: 锚点映射，将锚点名称映射到目标节点名称。这匹配GetNodeData输出格式。
- **Recognition**: 定义节点如何在屏幕上识别目标。
- **Action**: 定义当识别成功时执行的动作。
- **Next**: 指定可能执行的下一个节点列表。
- **RateLimit**: 设置识别尝试之间的最小时间间隔（毫秒），默认为1000。
- **Timeout**: 设置等待识别的最大时间（毫秒），默认为20000。
- **OnError**: 指定当识别超时或动作执行失败时执行的节点。
- **Inverse**: 反转识别结果，即识别失败时视为成功。
- **Enabled**: 确定节点是否处于活动状态，默认为true。
- **MaxHit**: 设置节点的最大命中次数，达到后不再执行。
- **PreDelay**: 在动作执行前的延迟时间（毫秒），默认为200。
- **PostDelay**: 在动作执行后的延迟时间（毫秒），默认为200。
- **PreWaitFreezes**: 在动作执行前等待屏幕稳定。
- **PostWaitFreezes**: 在动作执行后等待屏幕稳定。
- **Repeat**: 设置节点重复执行的次数，默认为1。
- **RepeatDelay**: 在重复执行之间的延迟时间（毫秒），默认为0。
- **RepeatWaitFreezes**: 在重复执行之间等待屏幕稳定。
- **Focus**: 指定自定义焦点数据。
- **Attach**: 为节点提供附加的自定义数据。

## 识别配置（Recognition）

`Recognition`结构体定义了节点的识别配置，采用统一的参数类型和接口设计。新的系统移除了旧的功能选项模式，改用直接的参数对象。

### 识别类型

- **DirectHit**: 不进行实际识别，总是成功。
- **TemplateMatch**: 模板匹配，通过图像模板进行识别。
- **FeatureMatch**: 特征匹配，提供更好的透视和尺度不变性。
- **ColorMatch**: 颜色匹配，基于颜色空间进行识别。
- **OCR**: 光学字符识别，用于识别文本。
- **NeuralNetworkClassify**: 神经网络分类，对固定位置的图像进行分类。
- **NeuralNetworkDetect**: 神经网络检测，检测任意位置的对象。
- **And**: 逻辑与组合，要求所有子识别都成功。
- **Or**: 逻辑或组合，任一子识别成功即可。
- **Custom**: 自定义识别，使用用户注册的识别器。

### 统一参数接口

新的识别系统采用统一的`RecognitionParam`接口，所有识别类型的参数都实现了这个接口：

```go
type RecognitionParam interface {
    isRecognitionParam()
}
```

### 参数对象使用方式

每个识别类型都有对应的参数结构体，直接作为参数传入构造函数：

```go
// 模板匹配示例
rec := RecTemplateMatch(TemplateMatchParam{
    ROI: Target{X: 100, Y: 100, Width: 200, Height: 200},
    Template: []string{"template.png"},
    Threshold: []float64{0.7},
})

// OCR示例
rec := RecOCR(OCRParam{
    ROI: Target{X: 0, Y: 0, Width: 300, Height: 50},
    Expected: []string{"Hello"},
    Threshold: 0.3,
})
```

### 组合识别

And和Or识别类型支持子识别的组合：

```go
// 与组合：所有条件都必须满足
rec := RecAnd(
    Ref("NodeA"),                    // 引用其他节点
    Inline(RecDirectHit(), "sub1"),  // 内联识别
)

// 或组合：任一条件满足即可
rec := RecOr(
    Ref("NodeB"),
    Inline(RecColorMatch(ColorMatchParam{
        ROI: Target{X: 100, Y: 100, Width: 200, Height: 200},
        Lower: [][]int{{0, 0, 0}},
        Upper: [][]int{{255, 255, 255}},
        Count: 1,
    })),
)
```

## 动作配置（Action）

`Action`结构体定义了节点的动作配置，同样采用了统一的参数类型和接口设计。新的系统移除了旧的功能选项模式，改用直接的参数对象。

### 动作类型

- **DoNothing**: 不执行任何操作。
- **Click**: 点击指定位置。
- **LongPress**: 长按指定位置。
- **Swipe**: 滑动操作。
- **MultiSwipe**: 多指滑动。
- **TouchDown**: 触摸按下。
- **TouchMove**: 触摸移动。
- **TouchUp**: 触摸释放。
- **ClickKey**: 点击虚拟键。
- **LongPressKey**: 长按虚拟键。
- **KeyDown**: 按下虚拟键。
- **KeyUp**: 释放虚拟键。
- **InputText**: 输入文本。
- **StartApp**: 启动应用。
- **StopApp**: 停止应用。
- **StopTask**: 停止当前任务链。
- **Scroll**: 滚动操作。
- **Command**: 执行命令。
- **Shell**: 执行Shell命令（仅限ADB控制器）。
- **Custom**: 自定义动作。

### 统一参数接口

新的动作系统采用统一的`ActionParam`接口，所有动作类型的参数都实现了这个接口：

```go
type ActionParam interface {
    isActionParam()
}
```

### 参数对象使用方式

每个动作类型都有对应的参数结构体，直接作为参数传入构造函数：

```go
// 点击动作示例
act := ActClick(ClickParam{
    Target: Target{X: 100, Y: 200},
    TargetOffset: Rect{X: 5, Y: 5, Width: 0, Height: 0},
    Contact: 0,
})

// 长按动作示例
act := ActLongPress(LongPressParam{
    Target: Target{X: 150, Y: 250},
    Duration: 1000 * time.Millisecond, // 使用time.Duration
    Contact: 0,
})

// 滑动动作示例
act := ActSwipe(SwipeParam{
    Begin: Target{X: 100, Y: 200},
    End: []Target{{X: 100, Y: 300}},
    Duration: []time.Duration{200 * time.Millisecond},
    EndHold: []time.Duration{0 * time.Millisecond},
    OnlyHover: false,
    Contact: 0,
})
```

### 多指滑动

MultiSwipe动作支持多指同时滑动：

```go
act := ActMultiSwipe(
    MultiSwipeItem{
        Starting: 0 * time.Millisecond,
        Begin: Target{X: 100, Y: 200},
        End: []Target{{X: 100, Y: 300}},
        Duration: []time.Duration{200 * time.Millisecond},
        Contact: 0,
    },
    MultiSwipeItem{
        Starting: 0 * time.Millisecond,
        Begin: Target{X: 150, Y: 250},
        End: []Target{{X: 150, Y: 350}},
        Duration: []time.Duration{200 * time.Millisecond},
        Contact: 1,
    },
)
```

## 流程控制

`NextItem`结构体定义了节点执行成功后的下一个节点信息。新的系统保留了功能选项模式，但简化了使用方式。

### NextItem属性

- **Name**: 目标节点的名称。
- **JumpBack**: 指示在此节点链完成后是否跳回父节点。
- **Anchor**: 指示此节点是否应设置为锚点。

### 功能选项使用

新的系统提供了简化的功能选项：

```go
// 添加下一个节点并启用跳回机制
node.AddNext("TaskA", WithJumpBack())

// 添加下一个节点并设置为锚点
node.AddNext("TaskB", WithAnchor())

// 同时设置多个属性
node.AddNext("TaskC", WithJumpBack(), WithAnchor())
```

### 锚点管理

新系统提供了更直观的锚点管理方法：

```go
// 设置锚点映射
node.SetAnchor(map[string]string{
    "anchor1": "NodeA",
    "anchor2": "NodeB",
})

// 设置特定锚点
node.SetAnchorTarget("anchor1", "NodeA")

// 添加锚点
node.AddAnchor("anchor1")

// 清除锚点
node.ClearAnchor("anchor1")

// 移除锚点
node.RemoveAnchor("anchor1")
```

## 错误处理与超时

错误处理和超时配置保持不变，继续使用直接的数值配置：

- **RateLimit**: 设置识别尝试之间的最小时间间隔（毫秒），默认为1000。
- **Timeout**: 设置等待识别的最大时间（毫秒），默认为20000。
- **OnError**: 指定当识别超时或动作执行失败时执行的节点列表。

配置方式保持简单直接：

```go
// 设置超时时间为30秒
node.SetTimeout(30 * time.Second)

// 设置识别间隔为2秒
node.SetRateLimit(2 * time.Second)

// 设置错误处理节点
node.SetOnError([]NextItem{
    {Name: "RetryNode"},
    {Name: "FallbackNode", JumpBack: true},
})
```

## Go API动态构建

新的API重构移除了旧的功能选项模式，改用更直接的参数对象方式。`NewPipeline`和`NewNode`函数保持不变，但节点配置方式更加直观。

### 基础API

```go
// 创建新的空流水线
pipeline := NewPipeline()

// 创建新节点
node := NewNode("Startup")
```

### 节点配置方法

新的系统提供了更直观的配置方法：

```go
// 设置识别配置
node.SetRecognition(RecTemplateMatch(TemplateMatchParam{
    ROI: Target{X: 100, Y: 100, Width: 200, Height: 200},
    Template: []string{"template.png"},
}))

// 设置动作配置
node.SetAction(ActClick(ClickParam{
    Target: Target{X: 150, Y: 250},
    Contact: 0,
}))

// 添加下一个节点
node.AddNext("TaskA")
node.AddNext("TaskB", WithJumpBack())
```

### 时间参数处理

新的系统使用Go的标准时间类型：

```go
// 使用time.Duration进行时间配置
node.SetTimeout(30 * time.Second)
node.SetRateLimit(2 * time.Second)
node.SetPreDelay(500 * time.Millisecond)
node.SetPostDelay(1 * time.Second)
```

### 集成示例

完整的流水线构建示例：

```go
func buildPipeline() *Pipeline {
    pipeline := NewPipeline()
    
    // 创建启动节点
    startup := NewNode("Startup")
    startup.SetRecognition(RecTemplateMatch(TemplateMatchParam{
        ROI: Target{X: 0, Y: 0, Width: 1920, Height: 1080},
        Template: []string{"startup_template.png"},
        Threshold: []float64{0.8},
    }))
    startup.SetAction(ActStartApp(StartAppParam{
        Package: "com.example.app",
    }))
    startup.AddNext("MainTask")
    
    // 创建主任务节点
    mainTask := NewNode("MainTask")
    mainTask.SetRecognition(RecOCR(OCRParam{
        ROI: Target{X: 100, Y: 100, Width: 300, Height: 50},
        Expected: []string{"Continue"},
        Threshold: 0.3,
    }))
    mainTask.SetAction(ActClick(ClickParam{
        Target: Target{X: 200, Y: 150},
    }))
    mainTask.AddNext("Complete", WithJumpBack())
    
    // 添加节点到流水线
    pipeline.AddNode(startup)
    pipeline.AddNode(mainTask)
    
    return pipeline
}
```

### 完整的Go代码示例

```go
package main

import (
    "fmt"
    "time"
    
    "github.com/MaaXYZ/maa-framework-go/v4"
)

func main() {
    // 初始化框架
    maa.Init()
    
    // 创建任务执行器
    tasker, err := maa.NewTasker()
    if err != nil {
        panic(err)
    }
    defer tasker.Destroy()
    
    // 创建资源管理器
    res, err := maa.NewResource()
    if err != nil {
        panic(err)
    }
    defer res.Destroy()
    
    // 加载资源
    res.PostBundle("./resource").Wait()
    tasker.BindResource(res)
    
    // 构建流水线
    pipeline := buildPipeline()
    tasker.BindPipeline(pipeline)
    
    // 运行任务
    detail, err := tasker.PostTask("Startup").Wait().GetDetail()
    if err != nil {
        panic(err)
    }
    
    fmt.Println("Task completed:", detail)
}

func buildPipeline() *maa.Pipeline {
    pipeline := maa.NewPipeline()
    
    // 启动节点
    startup := maa.NewNode("Startup")
    startup.SetRecognition(maa.RecTemplateMatch(maa.TemplateMatchParam{
        ROI: maa.Target{X: 0, Y: 0, Width: 1920, Height: 1080},
        Template: []string{"startup_template.png"},
        Threshold: []float64{0.8},
    }))
    startup.SetAction(maa.ActStartApp(maa.StartAppParam{
        Package: "com.example.app",
    }))
    startup.AddNext("MainTask")
    
    // 主任务节点
    mainTask := maa.NewNode("MainTask")
    mainTask.SetRecognition(maa.RecOCR(maa.OCRParam{
        ROI: maa.Target{X: 100, Y: 100, Width: 300, Height: 50},
        Expected: []string{"Continue"},
        Threshold: 0.3,
    }))
    mainTask.SetAction(maa.ActClick(maa.ClickParam{
        Target: maa.Target{X: 200, Y: 150},
    }))
    mainTask.AddNext("Complete", maa.WithJumpBack())
    
    pipeline.AddNode(startup)
    pipeline.AddNode(mainTask)
    
    return pipeline
}
```

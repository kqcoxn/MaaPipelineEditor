本地服务允许 MaaPipelineEditor 与您本地的其他应用程序进行实时通信，实现双向通信。

MaaPipelineEditor 提供了 **LocalBridge (LB)** 作为官方本地服务，目前主要用于本地文件管理，您也可以 [自行集成](/guide/other/protocol)。

::: tip
**本地服务是增量功能，非必须配置**，可仅通过[在线站点](https://mpe.codax.site/stable) 完成 Pipeline 编辑操作。

当然，在您**使用一行命令部署**本地服务后，将使您的体验有进一步的提升！
:::

## 功能概览

LocalBridge 目前提供以下功能：

- ✅ **本地文件管理**：扫描、打开、保存、新建、实时监听 Pipeline 文件等
- ✅ **字段辅助**：实时截图、内容识别、颜色提取等
- ✅ **流程级调试**：完整运行、从节点运行、单节点运行、仅识别、仅动作、固定图识别、追踪与产物查看等
- ✅ **AI 辅助编辑**：智能搜索、节点补全、MaaMCP 支持等

::: tip 参与建议

:::

## 下载与安装

### 方式一：一键安装（推荐）

您可以在任意系统上使用一行指令下载并安装 LocalBridge：

::: code-group

```shell [Windows (PowerShell, 推荐)]
irm https://raw.githubusercontent.com/kqcoxn/MaaPipelineEditor/main/scripts/install/install.ps1 | iex
```

```shell [Linux/macOS]
curl -fsSL https://raw.githubusercontent.com/kqcoxn/MaaPipelineEditor/main/scripts/install/install.sh | bash
```

:::

::: tip 提示

- 在使用任一终端安装后，在其他终端也可使用！Windows 系统推荐使用 PowerShell 安装
- 安装完成后，您可以在全局使用 `mpelb` 命令启动服务
- 一键安装会同时准备 MaaFramework 运行环境与原生 OCR 资源；下载体积较大，安装过程中会显示进度
- LocalBridge 支持自动检测更新，启动时会提示最新版本，更新指令与安装相同
  :::

::: warning 安装失败？

如果安装时提示「获取版本信息失败」，通常是因为 GitHub API 未认证请求频率限制（每 IP 每小时 60 次）。

解决方法：设置 `GITHUB_TOKEN` 环境变量后重试（Token 无需勾选任何权限）：

```shell [Linux/macOS]
export GITHUB_TOKEN="your_github_token"
curl -fsSL https://raw.githubusercontent.com/kqcoxn/MaaPipelineEditor/main/scripts/install/install.sh | bash
```

```shell [Windows (PowerShell)]
$env:GITHUB_TOKEN="your_github_token"
irm https://raw.githubusercontent.com/kqcoxn/MaaPipelineEditor/main/scripts/install/install.ps1 | iex
```

Token 获取地址：[GitHub Settings > Tokens](https://github.com/settings/tokens)，创建时无需勾选任何 scope。
:::

### 附属运行环境

一键安装脚本会在安装 `mpelb` 的同时，自动准备以下附属文件：

- **MaaFramework runtime**：从 [MaaFramework Releases](https://github.com/MaaXYZ/MaaFramework/releases) 下载当前平台最新包，并提取其中的 `bin` 目录
- **原生 OCR 资源**：下载 `ppocr_v6-small.zip`，并提取 `det.onnx`、`rec.onnx`、`keys.txt` 等模型文件

默认目录结构如下：

```text
mpelb 所在目录/
├── mpelb(.exe)
└── runtime/
    ├── maafw/
    │   └── bin/
    └── resource/
        └── model/
            └── ocr/
```

安装脚本的判断逻辑：

- `mpelb` 本体每次运行安装脚本都会下载最新版本并覆盖旧文件
- `runtime/maafw/bin` 已存在且非空时，不会重复下载 MaaFramework runtime
- `runtime/resource/model/ocr` 已存在且非空时，不会重复下载 OCR 资源
- 安装脚本不会修改 LocalBridge 的 `config.json`，也不会覆盖您手动设置过的 `maafw.lib_dir` / `maafw.resource_dir`

如需更新附属运行环境，请先删除 `mpelb` 所在目录下的 `runtime` 目录，或删除对应子目录后重新运行安装脚本。

::: tip 配置优先级
如果您没有手动配置 MaaFramework 或 OCR 路径，LocalBridge 会自动使用安装目录中的 `runtime/maafw/bin` 与 `runtime/resource`。如果您已经通过 `mpelb config set-lib` 或 `mpelb config set-resource` 配置过路径，且该路径仍然存在，则优先使用您的配置。
:::

### 方式二：手动下载

您可以在 [GitHub Release](https://github.com/kqcoxn/MaaPipelineEditor/releases) 页面下载对应平台的 LocalBridge 可执行文件：

- Windows: `mpelb-windows-amd64.exe`
- Linux: `mpelb-linux-amd64`
- macOS (Apple Silicon): `mpelb-darwin-arm64`

下载后，您可以在可执行文件目录下通过命令行运行可执行文件，或添加环境变量后全局使用。（需要以文件名启动，建议您将文件名改为 `mpelb`）

::: warning 注意
手动下载只包含 `mpelb` 可执行文件，不会自动准备 MaaFramework runtime 与 OCR 资源。若需要设备连接、截图、原生 OCR、流程调试等功能，请自行在可执行文件同目录准备上文所示的 `runtime` 目录，或使用 `mpelb config set-lib` / `mpelb config set-resource` 指向已有环境。
:::

## 启动服务

### 快速启动

在您的 Pipeline 根目录下（推荐在 `Asstes` 文件夹路径），打开终端并运行：

```bash
mpelb
```

这将使用默认配置启动 LocalBridge：

- **扫描目录**：当前工作目录
- **服务端口**：9066
- **日志级别**：INFO
- **配置存储**：系统用户数据目录

启动成功后，您将看到类似输出：

```
[14:32:15][INFO][Main] Local Bridge 启动中... 版本: v0.10.0
[14:32:15][INFO][Main] 运行模式: 本地模式
[14:32:15][INFO][Main] 数据目录: C:\Users\YourName\AppData\Roaming\MaaPipelineEditor\LocalBridge
[14:32:15][INFO][Main] 运行目录: D:\my-pipelines
[14:32:15][INFO][WebSocket] WebSocket 服务已启动: ws://localhost:9066
[14:32:15][INFO][FileService] 初始扫描完成，发现 12 个文件
[14:32:15][INFO][FileWatcher] 文件监听器已启动
```

### 自定义配置（可选）

#### 指定扫描目录

如果您的 Pipeline 文件不在当前目录，可以指定根目录：

```bash
mpelb --root D:/my-pipelines
```

或者使用相对路径：

```bash
mpelb --root ./pipelines
```

#### 修改服务端口

如果默认端口 9066 被占用，可以指定其他端口：

```bash
mpelb --port 9999
```

::: warning 注意
修改端口后，需要在编辑器的配置面板中同步修改端口号
:::

#### 调整日志级别

调试时可以提高日志详细程度：

```bash
mpelb --log-level DEBUG
```

可选的日志级别：`DEBUG` | `INFO`（默认） | `WARN` | `ERROR`

#### 组合使用

您可以组合多个参数：

```bash
mpelb --root ./pipelines --port 9066 --log-level INFO --log-dir ./logs
```

### 可视化配置管理

除了使用命令行管理配置，您也可以在 MaaPipelineEditor 的前端界面中可视化地查看和修改所有配置项：

1. **打开配置面板**：点击界面右上角的 **⚙️ 配置** 按钮
2. **修改配置**：在配置面板中找到「LocalBridge 配置」按钮并点击
3. **调整配置项**：在弹出的配置对话框中修改需要的配置
4. **保存配置**：点击「保存配置」后，LocalBridge 会自动热重载，立即应用新配置

::: tip 热重载支持
LocalBridge 支持配置热重载，修改并保存配置后会自动重新加载，无需手动重启服务。热重载会保持连接状态，不会中断您的工作流程。
:::

### 使用配置文件

#### 配置管理命令

LocalBridge 提供了便捷的配置管理命令：

**打开配置文件**：

```bash
mpelb config open
```

该命令会使用系统默认编辑器打开配置文件。

**设置 MaaFramework 库路径**（可选覆盖）：

```bash
mpelb config set-lib [path]
```

- 不带参数：交互式选择或输入路径
- 带参数：直接设置指定路径

**设置 OCR 资源路径**（可选覆盖）：

```bash
mpelb config set-resource [path]
```

- 不带参数：交互式选择或输入路径
- 带参数：直接设置指定路径

**查看配置信息**：

```bash
mpelb info
```

显示当前运行模式、配置文件位置、日志目录等信息。

::: tip 完整指令速查
如需查看 LocalBridge 所有命令行参数和子命令的完整清单，请参考 [指令汇总](/guide/server/commands)。
:::

## 连接 Editor

### 连接与配置

在 MaaPipelineEditor 界面右上角,您会看到 LocalBridge 连接按钮:

- **未连接状态**:显示「未连接本地服务」,按钮为灰色
- **连接中状态**:显示「连接服务中...」,按钮禁用并显示加载图标
- **已连接状态**:显示「MPE LocalBridge」,按钮为蓝色

点击按钮即可连接或断开本地服务。

**后续，您可以在配置界面勾选自动连接功能，在打开 Editor 后会自动尝试连接到本地服务！**

### 配置端口

如果您修改了 LocalBridge 的启动端口,需要在前端进行相应配置:

1. 点击右上角的 **⚙️ 配置** 按钮打开配置面板
2. 在「本地通信」区域找到「端口」配置项
3. 输入您的自定义端口号（默认 9066）
4. 修改后需要重新点击连接按钮

::: tip 自动连接
在配置面板的「本地通信」区域,您可以开启「自动连接」选项,下次打开编辑器时会自动尝试连接本地服务
:::

## 断开连接

如果需要停止使用 LocalBridge:

1. 在配置面板中点击 **断开** 按钮
2. 或者直接关闭 LocalBridge 的终端窗口

::: warning 注意
LocalBridge 不会自动重连，断开后需要手动点击「连接」按钮重新建立连接
:::

## 常见问题

### MaaFramework 路径配置

如果启用了 MaaFramework 功能但未配置路径，启动时会提示：

```
未配置 MaaFramework lib 路径，请运行以下命令进行配置：
  mpelb config set-lib
```

按照提示运行配置命令，选择或输入 MaaFramework Release 包的 `bin` 目录即可。

### 自动更新检测

LocalBridge 启动时会自动检测新版本（开发版除外）。如果有新版本可用，会显示：

```
══════════════════════════════════════════════════
🎉 发现新版本
══════════════════════════════════════════════════
   当前版本: v0.9.0
   最新版本: v0.10.0

   下载地址:
   https://github.com/kqcoxn/MaaPipelineEditor/releases/tag/v0.10.0
   快速更新指令:
   irm https://raw.githubusercontent.com/kqcoxn/MaaPipelineEditor/main/scripts/install/install.ps1 | iex
══════════════════════════════════════════════════
```

复制对应系统的更新命令即可快速更新到最新版本。

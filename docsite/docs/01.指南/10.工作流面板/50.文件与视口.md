视口是承载 Pipeline 文件的可交互窗口。

## 文件

MPE 的一个文件视口相当于一个 `.json` Pipeline 文件，您可以在左上角增删文件与修改文件名。

![](/images/core/文件.png)

文件名仅作为 MPE 文件标签的标记，与您工作目录的文件名没有相关性。

切换文件时，视口内的节点与连接会进行相应切换。

## 视口

节点与连接都会保存在对应的文件中，通过视口进行交互。

## 文件配置

您可以对每个文件进行不同的配置，它们在设置面板中，目前支持如下配置：

- `节点前缀`：对于文件内的每个节点，在编译时会加入统一的前缀，并以 `_` 连接。

节点配置会在编译时以节点的形式保存在 Pipeline 文件中。

![](/images/core/文件配置.png)

例如，如上配置会被编译为：

```json
{
  "$__mpe_config_文件1": {
    "$__mpe_code": {
      "prefix": "前缀1", // 前缀
      "filename": "文件1", // 文件名
      "version": "v0.5.5" // 导出时使用的 MPE 版本号
    }
  },
  "前缀1_节点1": {
    /* ... */
    "$__mpe_code": { "position": { "x": 3681, "y": 2504 } } // 节点在 MPE 中的坐标
  }
}
```

若您不希望导出配置，可以在设置中关闭。

## 跨文件搜索

当连接 LocalBridge 后，MPE 支持跨文件搜索功能，可以在所有已扫描的本地文件中搜索节点。

### 启用/禁用

在配置面板的「本地服务」区域，可以通过 **启用跨文件搜索** 开关控制此功能。默认开启。

关闭后，搜索工具仅在当前打开的文件中搜索节点。

### 文件夹过滤

当项目文件较多时，可以通过 **文件夹过滤** 配置项限制搜索和文件列表的范围。

**配置方式**：

在配置面板的「本地服务」区域，找到「文件夹过滤」输入框，填写需要包含的文件夹相对路径。

**格式**：

- 多个路径之间用逗号、分号或换行分隔
- 路径相对于 LocalBridge 的扫描根目录
- 留空表示不过滤（显示所有文件）

**示例**：

```
pipeline
pipeline/sub
resource/tasks
```

**作用范围**：

- 本地文件列表面板中显示的文件
- 跨文件搜索的候选范围

::: tip
文件夹过滤不影响 LocalBridge 的文件监听范围，仅影响前端的显示和搜索结果。
:::

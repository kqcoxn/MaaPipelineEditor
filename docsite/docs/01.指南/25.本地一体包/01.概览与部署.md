## 概览

**MaaPipelineExtremer**（简称 Extremer）是 MaaPipelineEditor 的一体化桌面客户端，将前端编辑器、LocalBridge 后端服务、MaaFramework 运行环境整合为单个可执行文件，提供**开箱即用**的本地编辑与调试体验。

### 核心特性

- ✨ **开箱即用**：无需手动安装依赖、配置环境或启动多个服务
- 🔒 **完全本地**：所有数据存储在本地，无需联网即可使用全部功能
- 🚀 **一键启动**：双击启动，自动初始化工作目录、端口分配和后端服务
- 🧩 **集成完整**：内置前端编辑器、文件管理、截图识别、流程调试等全部功能
- 🔄 **自动更新**：支持检查更新并一键下载最新版本
- 🖥️ **跨平台支持**：支持 Windows、macOS 和 Linux 系统

#### 工作目录

工作目录默认位于系统文档目录下的 `MaaPipeline` 文件夹：

- **Windows**: `C:\Users\用户名\Documents\MaaPipeline\`
- **macOS**: `~/Documents/MaaPipeline/`
- **Linux**: `~/MaaPipeline/`

工作目录结构：

```
MaaPipeline/
├── files/              # Pipeline 文件
├── templates/          # 自定义模板
├── images/             # 截图与图片资源
└── config.json         # 配置文件
```

## 下载与安装

访问 [GitHub Releases](https://github.com/kqcoxn/MaaPipelineEditor/releases) 页面，根据您的操作系统选择对应的安装包：

- **Windows**: `MaaPipelineExtremer-windows-x64.zip`
- **macOS**: `MaaPipelineExtremer-darwin-universal.zip`
- **Linux**: `MaaPipelineExtremer-linux-x64.tar.gz`

解压双击 exe 即用！

## 配置管理

### 在前端修改配置（推荐）

Extremer 支持在前端界面中可视化修改所有配置项：

1. **打开配置面板**

   - 点击界面左上角的 **⚙️ 配置** 按钮

2. **打开后端配置**

   - 在配置面板中找到"本地通信"部分
   - 点击"本地服务配置"下的 **打开配置** 按钮

3. **修改配置项**

   在弹出的配置对话框中，您可以 LocalBridge 的各个配置，推荐您首先修改以下配置：

   - **根目录**：Pipeline 文件扫描根目录

4. **保存并生效**
   - 点击 **保存配置** 按钮
   - 部分配置（如端口）需要重启应用才能生效
   - 配置文件自动保存到工作目录

::: tip 提示
Extremer 默认已在 `resources` 目录中包含基础的 MaaFramework 文件和 OCR 资源，通常无需额外配置。如果默认配置不可用，或希望自行指定 MFW/OCR 版本，请按上述步骤手动设置。
:::

## 目录结构

```
Extremer/
├── MaaPipelineEditor.exe  # 主程序
├── config/                # 配置文件目录
│   └── default.json       # 默认配置文件
├── resources/             # 资源目录
│   ├── mpelb.exe          # LocalBridge 可执行文件
│   ├── maafw/             # MaaFramework 库文件
│   │   └── bin/           # 运行时库文件（dll/so/dylib）
│   └── resource/          # OCR 模型等资源
└── logs/                  # 日志目录
    ├── mpelb.log          # LocalBridge 日志
    └── extremer.log       # 客户端日志
```

## 常见问题

### 应用无法启动

1. 检查是否被杀毒软件拦截
2. 确认目录路径不包含中文或特殊字符
3. 查看日志文件获取详细错误信息

### WebSocket 连接失败

1. 检查端口是否被占用
2. 在前端配置中修改 WebSocket 端口
3. 重启应用使配置生效

### OCR 识别不可用

1. 在前端配置面板中检查 MaaFramework 配置
2. 确认"启用 MaaFW"已勾选
3. 验证 Lib 目录和资源目录路径正确
4. 重启应用使配置生效

### 截图功能异常

1. 确认 MaaFramework 配置正确
2. 检查资源目录中是否存在 `model/ocr` 文件夹
3. 查看日志文件中的错误信息

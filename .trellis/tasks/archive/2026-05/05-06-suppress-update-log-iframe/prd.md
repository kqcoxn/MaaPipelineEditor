# iframe 模式启动时不主动弹出更新日志

## 背景

当前前端会在启动时比较 `mpe_last_version` 与当前版本；若版本变化，则自动弹出更新日志。
在 iframe/embed 模式下，这个自动弹窗会打断宿主页面的嵌入体验。

## 目标

当 URL 中带有 iframe/embed 模式标志时，应用启动后不应主动弹出更新日志。

## 非目标

- 不修改“手动打开更新日志”入口。
- 不修改普通模式下的自动弹出行为。
- 不调整更新日志内容、样式、存储 key 或版本比较策略。

## 需求

1. 在 iframe/embed 模式下，启动阶段跳过自动打开更新日志弹窗。
2. 普通模式下，现有自动弹出逻辑保持不变。
3. Header 中的更新日志图标入口继续可用，用户仍可手动打开更新日志。
4. 版本记录的本地持久化行为保持稳定，不引入额外副作用。

## 实现提示

- 优先复用现有 embed 判定能力，不新增重复的 URL 解析逻辑。
- 改动范围尽量限制在触发“自动弹出”的位置。

## 验收

- `?embed=true` 打开时，首次加载或版本变化时都不会自动弹出更新日志。
- 非 `embed` 模式下，版本变化时仍会自动弹出更新日志。
- 任意模式下，点击 Header 中的更新日志入口都可以正常打开弹窗。
